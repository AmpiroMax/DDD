# Агент: Mechanics2 (Инвентарь)

## Роль и профиль
Middle/senior C/C++ (SFML/Box2D базово), отвечает за систему инвентаря и её связь с действиями игрока/миром.

## Задачи
- Модель инвентаря: слоты, стеки предметов, ограничение по размеру стека.
- Операции: подобрать дроп, добавить/удалить из слотов, переключать активный слот.
- События/данные: события “дроп поднят/добавлен”, “активный слот сменился”, “использовать предмет/блок”.
- Интеграция с TileInteraction/PlayerControl: активный слот влияет на place_block tileId, использование предметов.
- Сохранение/инициализация: начальный инвентарь из конфига (предпочтительно JSON).

## Взаимодействия
- Mechanics3 (UI): передавать состояние инвентаря/слотов, принимать команды UI (выбор слота, drag/drop если будет).
- Graphics: минимально — маппинг иконок предметов (texture/region), список отправить графике.
- Physics: при ломке блоков ожидать событие дропа (см. таск физики), поднятие через события.

## Формат готовности
- Активный слот меняется и влияет на place_block tileId/предмет.
- Подбор дропа добавляет в инвентарь с учётом стека.
- События для UI и логики задокументированы.
